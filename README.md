# 🌧️ Прогноз опадів — Open-Meteo + ML

Проєкт передбачає наявність чи відсутність опадів на основі реальних метеорологічних даних сервісу [Open-Meteo](https://open-meteo.com).

---

## 📊 Які дані використовуються

Дані завантажуються щоденно (daily) через безкоштовне API Open-Meteo:

| Змінна | Опис |
|---|---|
| `precipitation_sum` | Сума опадів за день (мм) — **цільова змінна** |
| `rain_sum` | Сума дощу за день (мм) |
| `temperature_2m_max/min/mean` | Температура повітря (°C) |
| `windspeed_10m_max` | Максимальна швидкість вітру (км/год) |
| `relative_humidity_2m_max/min` | Відносна вологість (%) |
| `shortwave_radiation_sum` | Сумарна сонячна радіація (MJ/m²) |
| `et0_fao_evapotranspiration` | Еваpотранспірація (мм) |
| `precipitation_hours` | Кількість годин з опадами |

**Цільова змінна (rain_label):**
- `0` — опадів немає (`precipitation_sum == 0`)
- `1` — опади є (`precipitation_sum > 0`)

**Додаткові ознаки (feature engineering):**
- Лагові значення опадів та лейблу за 1-3 попередні дні
- Ковзне середнє температури за 7 днів
- Добовий діапазон температури
- Місяць закодований циклічно (sin/cos)

## 🤖 Моделі ML

У проєкті реалізовано та порівнюється два алгоритми класифікації:

1. **Random Forest** (`RandomForestClassifier`, 200 дерев)  
   — стійкий до перенавчання, добре обробляє нелінійні залежності

2. **Logistic Regression** (`LogisticRegression`)  
   — базова лінійна модель, інтерпретована та швидка

Обидві моделі навчаються зі `class_weight="balanced"` для компенсації дисбалансу класів (в деяких регіонах сухих днів значно більше, ніж дощових).

**Метрики оцінки:**
- Accuracy, Precision, Recall, F1-Score
- Матриця помилок (Confusion Matrix)
- Важливість ознак (для Random Forest)

## 📁 Структура проєкту

```
rainfall_project/
├── app.py              # Streamlit-застосунок (Варіант A — рекомендований)
├── main.py             # Скрипт без інтерфейсу (Варіант B — альтернатива)
├── fetch_data.py       # Окремий скрипт тільки для завантаження даних
├── weather_daily.csv   # Датасет (генерується автоматично або завантажується вручну)
├── requirements.txt    # Залежності Python
└── README.md           # Цей файл
```

### Що робить кожен файл

- **`app.py`** — головний Streamlit-застосунок. Складається з 3 кроків:
  1. Завантаження даних (кнопка "Отримати з Open-Meteo" або завантаження CSV)
  2. Навчання моделі + відображення метрик і матриці помилок
  3. Вибір дати → прогноз з ймовірністю

- **`main.py`** — аналог без Streamlit. При запуску послідовно: завантажує дані → готує ознаки → навчає дві моделі → виводить метрики → виводить прогноз для останнього дня.

- **`fetch_data.py`** — мінімальний скрипт лише для завантаження даних та збереження CSV. Корисний для попереднього наповнення датасету.

- **`weather_daily.csv`** — кешований датасет. Якщо файл вже є, `main.py` зчитує його, не роблячи HTTP-запит повторно.

## 🚀 Як запустити

### 1. Встановлення залежностей

```bash
pip install -r requirements.txt
```

### 2. Варіант A — Streamlit-застосунок

```bash
streamlit run app.py
```

Відкрийте у браузері: `http://localhost:8501`

**Послідовність дій у застосунку:**
1. Налаштуйте координати та дати в сайдбарі
2. Натисніть **"Отримати дані з Open-Meteo"**
3. Натисніть **"Навчити модель"**
4. Оберіть дату → натисніть **"Зробити прогноз"**

### 3. Варіант B — скрипт

```bash
python main.py
```

### 4. Тільки завантаження CSV

```bash
python fetch_data.py
```
